About

Armadillog is a web console log. It makes it easier and faster to sift through application logs etc.

Input a file or a piece of text, apply a set of filters and examine the result.

You can narrow the output down to the point where you see exactly what is relevant to your current need.

General

Armadillogs user inferface consist of:

  • topbar - the dark bar at the top, holding several buttons,
  • panels - areas toggled by respective buttons fron the topbar, containing specific functionalities,
  • content area - where you see the loaded content.

Input

First of all you should put some content into the console log. You can do it by:

  • loading a file through Input panel,
  • pasting from clipboard to a textarea through Input panel,
  • drag&drop-ing a file onto the content area.

Choosing an input method that involves a file will cause the console to automatically track it. E. g. you can choose a file that is being used as a log of a running application to get a live console log.

Ofcourse, the clear button in Input panel clears the content and stops the tracking. Also choosing a new input clears the old one.

Due to some browser environment constraints concerning performance, there are limitations on the input content size. If one of those limits is met, appropriate overflow from the beginning is skipped.

Filter

As you seek an answer for a specific question in your content, filters are the way to express it.

You can create multiple filters and they are applied to each line of the content in order of appearance, which you can adjust. A filter can be removed or muted.

There are several types of affecting the output by a filter. You can choose to:

  • show only lines containing matches,
  • show only matching fragments,
  • hide lines containing matches,
  • hide matching fragments,
  • highlight line containing matches,
  • highlight matching fragment.

A filter value can be expressed as:

  • a text,
  • a regular expression.

The matching is done case-insensitively.

You can highlight matches in the output by choosing a highlight type for your filter.

After creating or removing filters you have to hit the submit button to apply the changes. You can clear the filters. It is applied automatically.

The filters are persistent- stored in the browser storage, so you will have them next time you visit the tool.

Examine

So the filters are applied and you see only the relevant pieces of content. Is it still hard to read? Examine to the rescue!

Open the examine panel and choose a content line. What you see is two editable textareas- first for the raw content line and second for the filtered one. Bigger font size and ability to edit the content makes it easy to find what you are looking for.
